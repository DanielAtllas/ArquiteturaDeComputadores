Armazenamento - Estrutura de Dados

  • Andamento básico de uma arquitetura de dados

    • Até que um computador entenda e execute as instruções dada por um determinado programa, o mesmo passa por uma série de etapas, para que esses dados seja trabalhados em seu interno e gerem algum resultado, sendo preciso passar pelos seguintes passos:

    1. Linguagem de Programação
      2.
    2. Assembly
      3.
    3. Código de máquina ----- Sistemas Operacionais
      4.
    4. Hardware
      5.

    Sendo então, de forma simplificada, dado um comando/instrução em uma Linguagem de auto-nível, gerando um segundo script com o mesmo conteúdo, sendo esta etapa chamada de Linguagem de baixo-nível. Logo, esse script é trabalhado e sendo gerido pelo código/linguagem de máquina, onde juntamente com os Sistemas Operacionais, serão por fim, lidos e trabalhados pelos Hardwares.

  • Armazenamento (Registradores e Caches)

    • Estes, são os componentes de armazenamento encontrados em um processador, ocorrendo uma migração de dados/instruções entre a memória e o componente, dessa forma, ao se dado um comando em alguma linguagem, os dados a serem trabalhados, serão regidos por esta migração.

    • Memória - Cache

      • A migração entre a Mmeória - Cache, irá gerar um movimento de dados controlas pelo Hardware.


    • Memória - Registrador

      • Já a migração entre Memória - Registrador, se dá a movimentos de dados controlados por Programas.

    É bom saber também, que os computadores usados nestas operações, podem ser de 2 tipos, Rize ou Sisc

    • Computadores RISC

      • Possuem um conjunto de instruções simples, onde o seu número de registradores é reduzido.

    • Computadores SISC

      • Possuem um conjunto de instruções complexas, onde o seu número de registradores é aumentado.

  • Registradores

    • Cada CPU atual inclui registradores para vários propósitos, desde armazenar instruções sendo executadas ao armazenamento/manipulação de dados, até propósitos genéricos. Ou seja, podem ser de propósito especígico ou geral. Além disso, esses registradore costumam rodar na mesma velocidade que o resto da CPU, pois praticamente toda operação desempenhada pela CPU, envolve os registradores. Além disso, o número de registradores é dependente do design de cada CPU, pois não há como alterr sua quantia.

    • Registradores de uso Geral

      • São manipulados diretamente por instruções.

    • Registradores de uso específico

      • Armazena informações de controle do processador e do S.O.

  • Memória CACHE

    • Possui o objetivo de atuar como um buffer entre os registradores, sendo eles limitados e de alta velocidade se comparado a memória RAM. A memórua cahce é configurada de forma a sempre que os dados forem acessados pela memória RAM, o hardware de sistema primeiro verifica se os dados desejados estão no cache, sendo assim recuperados de maneira rápida e usados pela CPU.

    Em termos de capacidade e armazenamento, o cache é menor que a RAM, sendo necssário dividir o cache em seções que podem ser usadas para armazenar áreasa diferentes de memória RAM.

    • Níveis de CACHE

      • Os caches atualmente podem ser multi-níveis, podendo existir mais de um conjunto de cahces entre a CPU e a memória principal, onde em sua maioria, possuem 2 níveis:

      1. Cache L1
        2.

        Frequentemente localixado diretamente no chip da CPU e rodam na mesma velocidade que a CPU.

      2. Cache L2
        3.

        Frequentemente, é parte do módulo da CPU, rodando na mesma velocidade da CPU, porém, menor e mais lento que o cache L1.

    Portanto, o objetivo de todos os sub-sistemas de cache, é reduzir o tempo médio de acesso à memória RAM,

    Além disso, tem-se também, a memória RAM e os discos rígidos, onde a memória RAM, ou memória principal, representa o armazenamento eletrônico em massa nos computadores, sendo usada para armazenar dados e programas nquanto estão em uso. Sua operação básica é simples, onde em seu nível mais baixo possui Chips (circuitos integrados que executam a "memorização", tendo 4 níveis de conexões:

    1. Eletricidade
      2.

      Operam circuitos dentro do Chip;

    2. Dados
      3.

      Tranferem o I/O (entrada e saída) de dados;

    3. Leitura/Gravação
      4.

      Controle de recuperação/armazenamento de dados;

    4. Endereço
      5.

      Onde os dados devem ser lidos/gravados dentro do Chip.

    • Armazenamento de dados na RAM
      1. Os dados são apresentados às conexões de dados;

      2. O endereço no qual o dado deve ser armazenado é apresentado às conexões de endereço;

      3. A conexão leitura/gravação está ajustada no modo Write (gravação).
    • Recuperação de dados na RAM
      1. Os dados são apresentados às conexões de endereço;

      2. A conexão leitura/gravação está ajustada no modo Read (leitura).

      3. Os dados são acessados pelas conexões de dados.

    OBS: Todas essas operações ocorrem de maneira rápida (em nanosegundos), assim como todas as tecnologias abordadas até aqui, são de Natureza Volátil (Peridas quando a energia é desligada)

    Já os discos rígidos, são não voláteis, os quais são permanentes mesmo após a queda de energia, estão ocupando uma especial posição na arquitetura , tornando-os ideais para armazenamentos de longo prazo. Porém, saiba que não é poss´vel executar programas diretamente quando estão no disco rígido, precisando ser acessados primeiramente pela memória RAM.

    Além disso, por ser de natureza eletromecânica, possui uma lentidão de armazenamento e recuperação de dados quando coparado com os demais, possuindo então 4 diferentes fases para suas operações, sendo:

    1. Movimento do braço mecânico;
    2. Rotação do disco;
    3. Heads acessando/gravando dados;
    4. Transferência de dados para os eletrônicos de drive.
